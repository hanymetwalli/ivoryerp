# 01_PRD_Permissions_Frontend: واجهة نظام الاستئذان

## 1. Objective
بناء واجهة مستخدم (UI) باستخدام React للسماح للموظفين بتقديم طلبات الاستئذان، مع ربطها بالـ Backend API الذي تم إنشاؤه مؤخراً.

## 2. Core Features (Frontend)
1. **نموذج الطلب (Form):** يحتوي على الحقول التالية:
   - تاريخ الطلب (Date Picker).
   - وقت البداية (Time Input).
   - وقت النهاية (Time Input).
   - سبب الاستئذان (Textarea).
2. **التحقق اللحظي (Client-Side Validation):**
   - منع المستخدم من اختيار وقت نهاية يسبق وقت البداية قبل إرسال الطلب.
   - التأكد من تعبئة جميع الحقول.
3. **معالجة الاستجابة (Error Handling & UX):**
   - عند نجاح الطلب (200/201): إظهار رسالة نجاح (Toast) وتفريغ النموذج.
   - عند استلام خطأ 422 (تجاوز الرصيد): قراءة الـ JSON response وعرض رسالة واضحة للموظف توضح (الرصيد المسموح، الرصيد المستهلك، والمدة المطلوبة).

## 3. Tech Stack & Constraints
- استخدام `src/api/ivoryClient.js` لإرسال الطلب عبر `POST` إلى نقطة النهاية المخصصة (مثلاً `/api/permissions`).
- استخدام مكونات `Shadcn UI` الموجودة في `src/components/ui/` لتصميم النموذج (Input, Button, Label, Toast).
